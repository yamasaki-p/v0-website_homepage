# Website Structure

---

## 1. サイト全体の目的

### 目的

Webサイトは「この人に相談してみたい」と思わせ、接点を生む場として機能させる。

### 具体的な役割

| 役割 | 説明 |
|------|------|
| 認知形成 | 何をしている会社かを端的に伝える |
| 課題共感 | 訪問者が抱える課題を言語化し、共感を得る |
| 価値理解 | 提供価値と支援後の状態を具体的に伝える |
| 信頼獲得 | 実績・経験・アプローチで信頼を形成する |
| 接点創出 | 問い合わせ・メルマガ登録への導線を設計する |

### サイトのゴール

1. **問い合わせの獲得**: 相談・商談への接続
2. **メルマガ登録の獲得**: 継続的な接点形成
3. **専門性の認知**: 記事コンテンツを通じた信頼構築

## 2. 想定読者

### プライマリ読者

| 属性 | 詳細 |
|------|------|
| 役職 | 事業部長 / 部門長 / DX推進責任者 / 人材開発責任者 |
| 所属 | 大手企業の事業部門・DX部門・人材開発部門 |
| 状況 | 変革テーマやAI活用を推進する立場にあるが、進め方に課題を感じている |

### 読者の訪問時の心理状態

- 「変革を進めたいが、うまく進んでいない」
- 「AI活用を進めたいが、現場に定着しない」
- 「外部の力を借りたいが、誰に頼めばいいかわからない」
- 「戦略コンサルは高い・抽象的、SIerは実行だけ」

### 読者の情報ニーズ

| フェーズ | ニーズ |
|----------|--------|
| 認知段階 | 何をしている会社か知りたい |
| 検討段階 | 自分の課題を解決できるか判断したい |
| 比較段階 | 他社との違いを理解したい |
| 決定段階 | 相談してみたいと思える根拠がほしい |

### ページ一覧（5ページ + 動的ページ）

| 統合前 | 統合先 | 理由 |
|--------|--------|------|
| 実績・事例 | トップページ + 各サービスページ | 信頼形成はページ内で完結させる |
| 会社概要 | トップページ | 1ページで会社理解を完結させる |

## 4. 各ページの役割と内容

### 4.1 トップページ（`/`）

**役割**: サイト全体の入口。課題共感 → 価値理解 → サービス概観 → 信頼形成 → CTA の流れで構成する。会社概要・実績もこのページで伝える。

**構成セクション**:

| セクション | 内容 | 目的 |
|------------|------|------|
| ヒーロー | キャッチコピー + サブコピー + CTA | 課題提起と価値訴求 |
| 課題共感 | 読者が抱える状況を言語化 | 「自分ごと」と感じさせる |
| 提供価値 | 支援後の状態を提示 | 何が変わるかを伝える |
| サービス概要 | 2サービスの概観 + 各詳細への導線 | サービス理解と回遊促進 |
| 強み・差別化 | アプローチの特徴 | 他社との違いを示す |
| 実績 | 支援実績・代表的な事例（3件程度） | 信頼形成 |
| 会社概要 | 会社情報 + 代表プロフィール | 信頼補完 |
| 記事ピックアップ | 最新記事（3件程度） | 専門性の訴求 + 回遊促進 |
| メルマガ登録 | 登録フォーム（埋め込み） | 継続接点の獲得 |
| CTA | 問い合わせへの導線 | コンバージョン |

**強み・差別化で伝えること**:
- 戦略だけでも、進行管理だけでもない
- 上流（構想・意思決定）と下流（現場・実装）の断絶を埋める
- 「決める」ところから「動かす」ところまで一貫して支援

**実績セクションの構成**:
- 支援社数・プロジェクト数（数値）
- 支援業界一覧
- 代表的な事例カード（3件程度）
  - 業界 / 支援テーマ / 成果概要

**会社概要セクションの構成**:
- 会社名・代表者・設立・所在地・事業内容
- Mission / Vision（簡潔に）
- 代表プロフィール（経歴・専門領域）

---

### 4.2 戦略PMO支援（`/services/strategy-pmo`）

**役割**: 戦略PMO支援サービスの詳細を伝え、相談への動機を形成する。

**構成セクション**:

| セクション | 内容 |
|------------|------|
| ヒーロー | サービス名 + 一言説明 + CTA |
| 課題 | このサービスが解決する課題（5個程度） |
| 提供価値 | 支援によって何が変わるか（3-4個） |
| 強み・差別化 | 戦略PMO支援における独自の強み |
| 支援内容 | 具体的に何をするか |
| 成果物 | 代表的なアウトプット一覧 |
| 支援後の状態 | Before → After の形式で提示 |
| 実績・事例 | 戦略PMO支援の実績（該当あれば） |
| CTA | 問い合わせへの導線 |

**課題（掲載例）**:
1. 構想はあるが、現場が動ける単位に落ちていない
2. 変革テーマが施策群として存在し、優先順位が曖昧
3. 意思決定の論点が整理されないまま会議が増える
4. 実行フェーズで複数部門・複数ベンダーにより複雑化する
5. 正論だけでは進まず、組織の力学を踏まえた進め方が必要

**強み（掲載例）**:
- 事業部長レイヤーの意思決定支援経験（戦略コンサル出身）
- 実行可能な形に落とし込む現場理解（事業会社での新規事業・DX経験）
- 複雑な実行体制を動かす横断推進力（50名以上のマネジメント経験）

**実績・事例セクションの構成**:
- 戦略PMO支援の支援実績数
- 代表的な事例（2-3件）
  - 業界 / 企業規模
  - 支援テーマ
  - 課題 → 成果

### 4.3 AI活用支援（`/services/ai-enablement`）

**役割**: AI活用支援サービスの詳細を伝え、相談への動機を形成する。

**構成セクション**:

| セクション | 内容 |
|------------|------|
| ヒーロー | サービス名 + 一言説明 + CTA |
| 課題 | このサービスが解決する課題（5個程度） |
| 提供価値 | 支援によって何が変わるか（3-4個） |
| 強み・差別化 | AI活用支援における独自の強み |
| 支援内容 | 具体的に何をするか |
| 成果物 | 代表的なアウトプット一覧 |
| 支援後の状態 | Before → After の形式で提示 |
| 実績・事例 | AI活用支援の実績（該当あれば） |
| CTA | 問い合わせへの導線 |

**課題（掲載例）**:
1. AI活用が現場業務に定着せず、一過性で終わる
2. AI活用が一部の先進人材や一部の部署に閉じる
3. 現場業務に即したユースケース設計が難しい
4. AIテーマが乱立するが、成果につながるものに絞れない
5. AIエージェントは期待が大きい一方、設計と導入が難しい

**強み（掲載例）**:
- 業務理解を前提としたAI活用設計
- 個人技で終わらせず組織能力として定着させる支援
- AIエージェントを業務実装に近づける伴走力

**実績・事例セクションの構成**:
- AI活用支援の支援実績数
- 代表的な事例（2-3件）
  - 業界 / 企業規模
  - 支援テーマ
  - 課題 → 成果

### 4.4 記事一覧（`/articles`）・個別記事（`/articles/[slug]`）

**役割**: 専門性を示し、継続的な接点を形成する。SEO流入の受け皿にもなる。

**記事一覧ページの構成**:

| セクション | 内容 |
|------------|------|
| ヘッダー | ページタイトル + 概要説明 |
| カテゴリフィルタ | 記事カテゴリでの絞り込み |
| 記事一覧 | 記事カード（タイトル・概要・日付・カテゴリ） |
| メルマガ登録 | フォーム埋め込み |

**個別記事ページの構成**:

| セクション | 内容 |
|------------|------|
| 記事ヘッダー | タイトル・公開日・カテゴリ |
| 本文 | 記事コンテンツ |
| 著者情報 | 執筆者プロフィール |
| 関連記事 | 同カテゴリの記事（3件程度） |
| メルマガ登録 | フォーム埋め込み |
| CTA | サービスページ or 問い合わせへの導線 |

**記事カテゴリ案**:
- 戦略PMO / 変革推進
- AI活用 / AIエージェント
- 組織・マネジメント
- 事例・インサイト

---

### 4.5 お問い合わせ（`/contact`）

**役割**: 相談・商談への接点を生む最終導線。

**構成セクション**:

| セクション | 内容 |
|------------|------|
| ヘッダー | ページタイトル + 相談への心理的ハードルを下げる一言 |
| フォーム | 名前・会社名・メールアドレス・相談内容 |
| 補足情報 | 返信目安・対応可能な相談内容 |

**フォーム項目**:
- 氏名（必須）
- 会社名（必須）
- メールアドレス（必須）
- 電話番号（任意）
- 相談内容（必須・自由記述）

**注意点**:
- 項目は最小限に
- 「無料相談」「資料請求」などの表現は避ける
- 「まずは相談から」「話を聞いてみる」のトーンで

---

### 4.6 メールマガジン登録（`/newsletter`）

**役割**: 継続的な接点を形成し、関係構築を図る。

**構成セクション**:

| セクション | 内容 |
|------------|------|
| ヘッダー | タイトル + メルマガの価値提案 |
| 配信内容 | 何を届けるかの説明（3-4点） |
| 登録フォーム | メールアドレス入力 |
| 補足 | 配信頻度・解除方法 |

**価値提案の例**:
- 変革推進・AI活用に関する実践的なインサイト
- 最新記事のお知らせ
- イベント・セミナー情報（実施時）

**注意点**:
- 登録のハードルを下げる（メールアドレスのみ）
- 営業色を出さず、価値提供を前面に
- トップページ・記事ページにも埋め込みフォームを設置

## 5. トップページで伝えるべき範囲

### 伝えること

| 要素 | 内容 | 詳細度 |
|------|------|--------|
| 課題共感 | 読者が抱える状況 | 代表的な課題を3-4個 |
| 提供価値 | 何が変わるか | 概要レベル |
| サービス | 2サービスの存在と概要 | 各サービス1-2文 |
| 強み | 差別化ポイント | 3点程度 |
| 実績 | 信頼の根拠 | 数値 + 代表事例3件 |
| 会社概要 | 会社情報 + 代表プロフィール | 簡潔に |
| 記事 | 専門性の訴求 | 最新3件のピックアップ |

### 伝えないこと（サービスページに委ねる）

- 各サービスの詳細な課題リスト
- 支援内容の詳細
- 成果物一覧
- Before → After の詳細
- サービス別の詳細事例

### 設計原則

- トップページで「会社理解」と「信頼形成」を完結させる
- 詳細を知りたい人はサービスページへ誘導
- 1ページで全体像が把握できる構成に

## 6. サービスページに分けるべき内容

### 戦略PMO支援ページに記載

| 項目 | 内容 |
|------|------|
| 課題詳細 | 変革推進における5つの課題 |
| 提供価値 | 構想整理・施策設計・実行推進の具体 |
| 強み | 戦略PMO支援固有の強み |
| 支援内容 | 具体的な活動内容 |
| 成果物 | 重点施策ポートフォリオ、推進ロードマップ等 |
| Before → After | 支援前後の状態変化 |
| 実績・事例 | 戦略PMO支援に関連する事例（2-3件） |

### AI活用支援ページに記載

| 項目 | 内容 |
|------|------|
| 課題詳細 | AI活用における5つの課題 |
| 提供価値 | 業務接続・組織定着・横展開の具体 |
| 強み | AI活用支援固有の強み |
| 支援内容 | 具体的な活動内容 |
| 成果物 | ユースケース一覧、研修プログラム、活用ガイド等 |
| Before → After | 支援前後の状態変化 |
| 実績・事例 | AI活用支援に関連する事例（2-3件） |

### 分割の理由

- 読者は自分に関係するサービスの詳細を知りたい
- トップページが長くなりすぎると離脱する
- サービスごとに課題・強み・事例が異なるため、個別に訴求する方が効果的

## 8. 主要導線とCTAの流れ

### 主要導線パターン

```
【パターン1: 直接問い合わせ】
トップページ → 問い合わせ

【パターン2: サービス理解後に問い合わせ】
トップページ → サービスページ → 問い合わせ

【パターン3: 記事からの流入】
記事 → トップページ or サービスページ → 問い合わせ

【パターン4: メルマガ登録】
トップページ or 記事 → メルマガ登録
```

### CTA配置方針

| 配置箇所 | CTA種別 | 目的 |
|----------|---------|------|
| グローバルヘッダー | 問い合わせボタン | 常時アクセス可能に |
| トップページ・ヒーロー | 問い合わせボタン | ファーストビューでのCV |
| トップページ・末尾 | 問い合わせボタン | 読了後のCV |
| トップページ・中盤 | メルマガ登録フォーム | 継続接点の獲得 |
| サービスページ・ヒーロー | 問い合わせボタン | サービス関心層のCV |
| サービスページ・末尾 | 問い合わせボタン | 詳細理解後のCV |
| 記事ページ・末尾 | メルマガ登録 + 問い合わせ | 継続接点 + CV |
| フッター | 問い合わせリンク | 常時アクセス可能に |

### CTA文言

| 用途 | 推奨文言 | 避ける文言 |
|------|----------|------------|
| 問い合わせ | 「相談する」「話を聞いてみる」 | 「無料相談」「今すぐお問い合わせ」 |
| サービス詳細 | 「詳しく見る」「サービスを見る」 | 「詳細はこちら」 |
| メルマガ | 「登録する」「受け取る」 | 「購読する」 |
